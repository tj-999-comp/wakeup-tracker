# 週次レポートメール実装ドキュメント

## 概要
このドキュメントは、Wakeup Tracker における週次レポートメール機能の実装内容を説明するものです。

現在の実装では、`Report.gs` で生成した週次集計結果をもとに、HTML 形式のレポートメールを作成し、Google Apps Script から自動送信できるようにしています。  
また、Apps Script の時間主導トリガーを設定し、毎週月曜日に自動送信される構成にしています。

---

## この機能でできること
現在の週次レポートメール機能では、以下が可能です。

- 前週月曜〜日曜の起床記録を集計する
- 集計結果を HTML メールとして整形する
- 自分宛にテスト送信する
- Google Drive にプレビュー用 HTML を保存する
- 毎週月曜日に自動送信する

---

## 関連ファイル
この機能は主に以下のファイルで構成されています。

| ファイル | 役割 |
| --- | --- |
| `Report.gs` | 週次集計ロジック |
| `Mail.gs` | メール本文生成、送信処理、プレビュー処理 |
| `weekly-report-template.html` | HTML メールテンプレート |

---

## 全体の流れ
週次レポートメールの処理は、以下の流れで動きます。

1. 前週の対象期間を取得する
2. `records` シートから対象週の記録を取得する
3. 集計結果を生成する
4. 集計結果をテンプレートに流し込む
5. HTML メール本文を生成する
6. プレーンテキスト本文も生成する
7. メール送信する
8. 毎週月曜日にトリガーで自動実行する

---

## `Mail.gs` の役割
`Mail.gs` では、主に以下の処理を担当しています。

- HTML メール本文の生成
- テキスト本文の生成
- テスト送信
- 本番送信
- プレビュー保存

`Report.gs` が「集計結果を返す」役割であるのに対し、`Mail.gs` は「その集計結果を見やすいメールとして出力する」役割を持ちます。

---

## 実装されている主な関数

### sendWeeklyReportEmail()
週次レポートメールを送信する本体の関数です。

### 役割
- 集計結果を取得する
- メール件名を作成する
- HTML 本文を生成する
- プレーンテキスト本文を生成する
- `MailApp.sendEmail()` で送信する

### 送信内容
- 宛先: 設定したメールアドレス
- 件名: 対象期間付きの週次レポート件名
- 本文:
  - プレーンテキスト本文
  - HTML 本文

### 件名例
```text
Wakeup Tracker 週次レポート (2026-03-09 ~ 2026-03-15)
```

---

### buildWeeklyReportHtml(stats)
集計結果をもとに、HTML メール本文を生成する関数です。

### 処理内容
1. `weekly-report-template.html` を読み込む
2. テンプレートへ表示用データを渡す
3. HTML として評価する
4. 文字列として返す

### 補足
Apps Script の `HtmlService.createTemplateFromFile()` を利用しています。  
テンプレートファイルを分けることで、HTML とロジックの責務を分離しています。

---

### buildWeeklyReportViewModel(stats)
集計結果を、テンプレートで扱いやすい表示用データへ変換する関数です。

### この関数を使う理由
テンプレート内で複雑な条件分岐や文字列整形をやりすぎると、HTML が読みにくくなります。  
そのため、テンプレートへ渡す前に Apps Script 側で整形済みデータを作ります。

### 変換している内容
- タイトル
- 対象期間表示
- 記録日数
- 総記録数
- 平日平均
- 休日平均
- 全体 earliest / latest の表示文字列
- 平日 earliest / latest の表示文字列
- 休日 earliest / latest の表示文字列
- 日別一覧の表示用配列
- バッジ用の CSS クラス
- バッジ表示ラベル

---

### buildWeeklyReportPlainText(stats)
HTML メールを受信できない環境でも最低限内容が読めるよう、プレーンテキスト本文を生成する関数です。

### 含めている内容
- 対象期間
- 記録日数
- 総記録数
- 平日平均
- 休日平均
- 全体 / 平日 / 休日 の earliest / latest
- 日別一覧

### 補足
`MailApp.sendEmail()` では、`htmlBody` と `body` を両方指定できます。  
そのため、HTML 表示に対応しない場合でも基本情報は確認できます。

---

### formatRecordSummaryText(record)
最早 / 最遅データを文字列へ整形する補助関数です。

### 例
```text
2026-03-13（金） 07:01 [weekday]
```

### 用途
- テキスト本文
- ビューモデル生成時の表示用テキスト

---

### previewWeeklyReportHtml()
生成した HTML 本文をログへ出力し、内容を確認するための関数です。

### 用途
- HTML 文字列の確認
- レイアウト調整前の簡易確認
- Drive 保存前の出力確認

---

### saveWeeklyReportPreviewToDrive()
生成した HTML を Google Drive に `.html` ファイルとして保存する関数です。

### 用途
- ブラウザでプレビュー確認する
- レイアウト確認する
- メール送信前に HTML の見た目を確認する

### 出力内容
- `weekly-report-preview.html`

### 動作
ファイル作成後、Drive 上の URL をログへ出力します。

---

## `weekly-report-template.html` の役割
`weekly-report-template.html` は、週次レポートの HTML テンプレートです。

### デザイン方針
今回のテンプレートは以下を意識しています。

- シンプルで見やすい
- カード風レイアウト
- メールクライアントで崩れにくい
- 平日 / 休日の違いが視覚的にわかる
- 週次レポートらしいまとまり感がある

### 主な構成
- ヘッダー
- 今週のサマリー
- 全体 / 平日 / 休日 の earliest / latest
- 日別一覧テーブル
- フッター

---

## テンプレートの表示内容

### 1. ヘッダー
最上部にタイトルと対象期間を表示します。

表示内容:
- `Wakeup Tracker Weekly Report`
- 対象期間

---

### 2. 今週のサマリー
サマリーカードとして以下を表示します。

- 記録日数
- 総記録数
- 平日平均
- 休日平均

これにより、レポートを開いてすぐ概要を把握できます。

---

### 3. 最早 / 最遅
以下の 3 観点で表示します。

- 全体
- 平日
- 休日

それぞれについて
- 最早
- 最遅

を表示することで、起床傾向を比較しやすくしています。

---

### 4. 日別一覧
日別一覧テーブルでは、以下の列を表示します。

- 日付
- 曜日
- 起床時刻
- 区分

区分にはバッジを付けており、視覚的に `weekday` / `holiday` を判別しやすくしています。

---

## CSS デザインの意図
��ンプレート内の CSS は、以下の意図で設計しています。

### 背景色
- 全体背景: 薄いグレー系
- コンテナ背景: 白
- 情報カード: 薄い青系

### 強調
- ヘッダーはグラデーション
- サマリー数値は大きめフォント
- バッジで平日 / 休日を色分け

### 読みやすさ
- 行間を十分に確保
- テーブルは罫線を控えめに
- セクションごとに余白をしっかり取る

---

## HTML 生成方法
HTML の生成には Apps Script のテンプレート機能を使っています。

### 使っている仕組み
- `HtmlService.createTemplateFromFile()`
- `template.evaluate().getContent()`

### 方針
HTML 内に直接集計ロジックを書かず、あらかじめ `buildWeeklyReportViewModel()` で表示用データに整形してからテンプレートへ渡す形を採用しています。

これにより、
- テンプレートが読みやすい
- 変更しやすい
- データ構造が明確
というメリットがあります。

---

## メール送信方法
メール送信には `MailApp.sendEmail()` を使用しています。

### 指定している内容
- `to`: 宛先メールアドレス
- `subject`: 件名
- `body`: プレーンテキスト本文
- `htmlBody`: HTML 本文
- `name`: 差出人表示名

### 実装上のポイント
HTML 本文だけでなく、プレーンテキスト本文も設定しています。  
これにより、メールクライアント側の事情で HTML が制限される場合でも、最低限内容は確認できます。

---

## 宛先設定
送信先は `sendWeeklyReportEmail()` 内の `REPORT_RECIPIENT` で指定しています。

### 例
```javascript
const REPORT_RECIPIENT = 'your-address@example.com';
```

### 運用方針
- 個人利用のため固定メールアドレスを設定
- テスト送信時も同じ宛先で確認
- トリガー実行時も同じ宛先へ送信

---

## プレビュー確認方法
実装時は、いきなり本番送信せず以下の方法で確認できます。

### 方法1: ログ出力
`previewWeeklyReportHtml()` を使って HTML をログへ出力する

### 方法2: Drive 保存
`saveWeeklyReportPreviewToDrive()` を使って HTML ファイルを Drive に保存し、ブラウザで確認する

### 方法3: テストメール送信
`sendWeeklyReportEmail()` を手動実行して、自分宛に送信して見た目を確認する

---

## トリガー設定
この実装では、Apps Script の時間主導トリガーを使って毎週月曜日に自動送信する設定を行っています。

### 設定内容
- 実行関数: `sendWeeklyReportEmail`
- イベントのソース: 時間主導型
- タイプ: 週ベースのタイマー
- 曜日: 月曜日
- 時間帯: 任意の朝の時間帯

### 運用イメージ
- 月曜日の朝に前週分のレポートを送信
- 集計対象は前週月曜〜日曜
- その週の振り返りとして確認する

## 実装時に意識したポイント

### 1. ロジックとテンプレートを分離する
HTML の見た目と Apps Script の処理ロジックを分けることで、今後の修正がしやすくなるようにしています。

### 2. テンプレート側のロジックを増やしすぎない
表示用文字列や CSS クラスなどは事前に ViewModel で組み立て、テンプレートをできるだけシンプルに保っています。

### 3. メールクライアントで崩れにくい構成にする
複雑すぎるレイアウトを避け、テーブルベースで比較的安定した表示になる構成を採用しています。

### 4. 手動確認しやすいようにする
ログ出力、Drive 保存、テスト送信の 3 ルートを用意し、見た目確認しやすいようにしています。
